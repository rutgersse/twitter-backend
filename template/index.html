Machine Learning Procedure (SVM)

Previously we did naive adaptive learning approach

Theoretically well motivated algorithm: developed from Statistical Learning Theory (Vapnik & Chervonenkis) since the 60s.

Empirically good performance: successful applications in many fields (bioinformatics, text, image recognition, . . . )

basic idea of SVM in layman terms find the separation between two set of values so that margin from set of values to separation plain become maximum impose more variable that affect ours analysis

What we did

  • trained first
  • 5000 + supervised tweet
  • trained it
  • then processed 20k + tweets
  • collect accurate tweets
  • those accurate tweets we monitored again and sampled them ( human corrected tweets )
  • merge those tweets to initial 5000 tweets
  • train the model again

More Analysis

  • So here we take tokens of the tweets as input
  • Every token has its own semantic value based on semantic analysis we did (i.e running 9, precidency -7 based on the analysis how the occurance of those words in correct and incorrect tweets)
  • so now every tweet we have some set of values
  • we impleted SVM and based on we calculated the support vector
  • Everytime we run this that produce different accurate vector
  • so sometimes when words are too much interrelated might produce incorrect result thats what we found after seeing analysed tweets
  • moved to non linear instead of linear
  • we used Kernal method
  • Kernal method is used for finding and studying general types of relations (for example clusters, rankings, principal components, correlations, classifications) in datasets -> And thats what exactly we want
  • so we created cluster based of semantic orientation based on the rating schema and also do classifications
